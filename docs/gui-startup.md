# Старт GUI

Этот документ описывает внутренний сценарий запуска GUI. В корневом README оставлено только краткое архитектурное описание, а подробности старта вынесены сюда.

## Общая схема

Старт GUI разделен на точку входа и UI-слой. `main.py` только настраивает процесс, создает `QApplication`, получает ключ сессии и открывает `NavigationGUI`.

Стартовый splash screen находится в `Navigation_Bot.gui.widgets.startup_splash.StartupSplash`. Его progress bar двигается от фактических этапов запуска, а не от таймера.

## Участники запуска

- `main.py` - точка входа приложения;
- `NavigationGUI` - главное окно;
- `NavigationGUI.update_startup_status(...)` - единая точка обновления стартового статуса;
- `StartupController` - последовательность этапов инициализации UI;
- `AppServices` - composition root GUI;
- `g.loading.show(...)` - общий механизм отображения прогресса и загрузки.

## Последовательность

1. `main.py` создает `QApplication`.
2. `main.py` показывает `StartupSplash`.
3. Приложение читает конфигурацию и готовит подключение к API.
4. Пользователь проходит авторизацию или используется ключ из конфигурации для отладки.
5. `main.py` создает `NavigationGUI`.
6. `NavigationGUI` восстанавливает настройки окна и таблиц.
7. `NavigationGUI` создает UI bridge, раскладку, контроллеры и сервисы.
8. `StartupController` запускает этапы локальной инициализации.
9. `AppServices` создает API-клиент, репозитории, прикладные сервисы, процессор навигации и UI-контроллеры.
10. Финальный статус `Готово` переводит splash на 100%.

## Источники обновления progress bar

Progress bar обновляется из нескольких мест:

- явные вызовы `splash.show(...)` и `splash.update(...)` в `main.py`;
- `NavigationGUI.update_startup_status(...)` во время сборки главного окна;
- этапы `StartupController`: локальные страницы, менеджеры, привязка сигналов, завершение;
- `AppServices` через `g.loading.show(...)`, который во время активного splash прокидывает статус в `update_startup_status(...)`.

Одинаковые сообщения нормализуются. Поэтому парные вызовы `update_startup_status(...)` и `loading.show(...)` для одного этапа не двигают progress bar дважды.

## Роль AppServices

`Navigation_Bot.gui.app.AppServices` является composition root GUI. Он собирает зависимости, которые нужны главному окну:

- конфигурацию базы/API;
- API-клиент;
- репозитории;
- сервисы Google Sheets;
- сервисы задач и навигации;
- менеджер таблицы;
- процессор обработки транспорта;
- контроллеры меню и пользовательских действий.

Такой подход удерживает создание зависимостей в одном месте и не размазывает инициализацию по виджетам.
