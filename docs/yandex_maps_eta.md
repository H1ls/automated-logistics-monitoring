# ETA из Яндекс.Карт

## Задача

При расчете прибытия нельзя складывать локальное время приложения с длительностью маршрута, если пункт назначения находится в другом часовом поясе.

Пример:

- сейчас в Москве: `02.07.2026 13:14`
- Яндекс.Карты показывают длительность: `10 ч 10 мин`
- пункт назначения: Владивосток
- Яндекс.Карты показывают ETA: `Прибытие 3 июля, в 06:14 (МСК +7)`
- дедлайн выгрузки: `03.07.2026 08:00` по локальному времени Владивостока

В московском времени через 10 часов будет еще `02.07.2026 23:24`, но для выгрузки важно локальное время назначения: `03.07.2026 06:24`. Поэтому для `успеет` нужно сравнивать ETA Яндекс.Карт с дедлайном выгрузки в той же локальной зоне.

## Текущая логика

`YandexMapsTab` читает из карточек маршрутов:

- `auto-route-snippet-view__duration`
- `auto-route-snippet-view__distance`
- `auto-route-snippet-view__arrival`

`MapsDataProcessor` обрабатывает эти данные:

- парсит длительность маршрута в минуты;
- считает среднее время и расстояние по всем авто-маршрутам;
- парсит ETA вида `Прибытие 8 июля, в 04:07 (МСК +7)`;
- выбирает год ETA, ближайший к дедлайну выгрузки;
- считает среднее ETA, если маршрутов несколько;
- парсит смещение относительно Москвы: `МСК +7`, `МСК +0`, `МСК -1`;
- записывает в `car["Маршрут"]` локальное ETA и смещение.

## Несколько маршрутов

Если Яндекс.Карты вернули несколько маршрутов, используются средние значения.

Пример:

| Длительность | ETA | Расстояние |
| --- | --- | --- |
| `20 ч 47 мин` | `Прибытие 3 июля, в 10:07 (МСК +0)` | `1740 км` |
| `1 дн. 2 ч 27 мин` | `Прибытие 3 июля, в 15:47 (МСК +0)` | `1780 км` |
| `22 ч 18 мин` | `Прибытие 3 июля, в 11:38 (МСК +0)` | `1750 км` |

Итог:

- средняя длительность: `1391 мин`, то есть `23 ч 11 мин`;
- среднее расстояние: `1757 км`;
- среднее ETA: `03.07.2026 12:31 (МСК +0)`.

## Результат в legacy-словаре

В `car["Маршрут"]` записываются поля:

```python
{
    "расстояние": "1757 км",
    "длительность": "1391 мин",
    "время прибытия": "03.07.2026 12:31",
    "успеет": True,
    "time_buffer": "3ч 29м",
    "buffer_minutes": 209,
    "timezone": "МСК +0",
    "timezone_offset_hours": 0,
}
```

`время прибытия` считается локальным временем пункта назначения. `timezone` показывает, относительно какой зоны Яндекс.Карты отдали ETA.

## Что еще стоит доработать

- Поддержать форматы ETA `сегодня` и `завтра`, если Яндекс.Карты начнут показывать их вместо даты.
- Явно форматировать отрицательный буфер, например `-1ч 46м`.
- Показывать `timezone` в таблице рядом с ETA, чтобы пользователь видел `08.07.2026 07:56 (МСК +7)`.
- Логировать предупреждение, если в нескольких маршрутах пришли разные смещения `МСК +N`.
